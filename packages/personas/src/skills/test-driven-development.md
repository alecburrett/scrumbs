# Test-Driven Development

Write the failing test first. Then write the minimum code to make it pass. Then refactor.

Red → Green → Refactor. Every time.

1. Write a failing test that specifies the desired behavior
2. Run it — confirm it fails (Red)
3. Write the minimum production code to pass the test
4. Run it — confirm it passes (Green)
5. Refactor with confidence
6. Repeat

Never write production code without a failing test. Never skip the refactor step.
